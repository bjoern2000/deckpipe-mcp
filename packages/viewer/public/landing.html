[ 01 ] what it does

A rendering engine for agents.

Your agent describes a deck as JSON. deckpipe handles layout, typography, and hosting.

feature/ 01

Layouts

Title, bullets, stats, quotes, timelines, SWOT, charts, galleries. Your agent chooses a layout. deckpipe handles the styling.

feature/ 02

MCP native

Remote server at /mcp for Claude.ai and Cursor. Local install via npx deckpipe-mcp for Claude Code.

feature/ 03

Theming

Any Google Font, any accent color, per-slide takeaways. Your agent changes one field; the whole deck re-renders.

feature/ 04

Live editing

PATCH with deep merge. insert, delete, move, replace operations. Your agent edits one bullet without re-sending the deck.

feature/ 05

Comments

You click any element to comment. Your agent reads the feedback, updates the deck, and replies.

feature/ 06

Image rehosting

Image URLs are re-hosted on deck creation, so slides keep working if the original URL goes down.

[ 03 ] connect

Connect deckpipe to your agent.

Three ways to do it, depending on which agent you use. They all give your agent the same tools.

01 //remote mcp  — url only, no install
claude.ai · cursor 
> https://deckpipe.com/mcp
02 //claude code
cli 
$ claude mcp add deckpipe -- npx deckpipe-mcp
03 //manual mcp config
json 
{
  "mcpServers": {
    "deckpipe": {
      "command": "npx",
      "args": ["-y", "deckpipe-mcp"]
    }
  }
}

[ 06 ] theming

Theming.

Theme settings live on the deck. Your agent can change them at any time and the whole deck re-renders.

heading_font any Google Font for headings
body_font any Google Font for body text
accent_color hex color, applied across the deck
key_takeaway per-slide highlight box

The icons_and_text layout takes any Lucide icon by name (e.g. "clock", "database") or any emoji.
